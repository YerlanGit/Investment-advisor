## 5. Стратегия: MCP и сервисы Anthropic — как это повлияет на проект

Сейчас бот использует Anthropic **минимально**: один `client.messages.create()` на отчёт, без кеша/инструментов/MCP/батча. Это и «дешёвая» победа по стоимости, и стратегический рычаг. Ниже — по приоритету влияния.

### 5.1 ✅ Prompt Caching — ВНЕДРЕНО (Sprint 2.4) · экономия до ~90% на префиксе
`SYSTEM_PROMPT.md` (~25 КБ) + RAG-контекст отправляются **целиком на каждый** запрос (а на пользователя их два — base и deep). Кешируя стабильный префикс (system + RAG), вы платите за него ~0.1× при повторных чтениях.
- **Эффект:** base+deep одного юзера и повторные прогоны → кеш-хиты; на объёме это кратно режет LLM-косты.
- **Как:** `cache_control: {type: "ephemeral"}` на последнем блоке system (и на RAG-блоке), волатильные данные (тикеры/дата) — после брейкпоинта. Проверять `usage.cache_read_input_tokens`.
- **Поддержка:** Haiku 4.5 и Sonnet 4.6 — мин. кешируемый префикс 4096 / 2048 токенов соответственно.

### 5.2 ✅ Structured Outputs — ВНЕДРЕНО (Sprint 2.5) · костыль `_repair_truncated_json` убран из пути
Сейчас нарратив возвращает JSON, который чинится вручную (`_repair_truncated_json`, обрезка по `max_tokens`). `output_config: {format: {type: "json_schema", schema: …}}` (через `client.messages.parse()` с Pydantic) **гарантирует** валидную схему.
- **Эффект:** удалить repair-хак, исключить класс «битый JSON → пустой отчёт».
- **Поддержка:** Haiku 4.5 / Sonnet 4.6 — да.

### 5.3 🔵 MCP — два направления (главный стратегический выбор)
**Model Context Protocol** — открытый стандарт Anthropic для подключения LLM к инструментам/данным. Для этого проекта он работает в **обе стороны**:

**(A) Бот как MCP-КЛИЕНT (потреблять данные).** Заменить/дополнить кастомные интеграции (Tradernet/FRED/SEC-EDGAR) готовыми финансовыми MCP-серверами (Alpha Vantage, Financial Modeling Prep, Financial Datasets, FactSet). Либо дать Claude доступ к свежим котировкам прямо в нарративе через `mcp_servers` (MCP-коннектор в Messages API) или SDK-хелперы `anthropic.lib.tools.mcp`.
- **Эффект:** шире покрытие активов/фундаментала меньшими силами; нарратив может «дотягивать» свежие факты.
- **Риск:** не отдавать LLM торговые/исполнительные MCP (Alpaca) без жёсткого human-in-the-loop.

**(B) Бот как MCP-СЕРВЕР (отдавать аналитику) — это и есть ваш ров.** Обернуть MAC3-движок (Эйлер-декомпозиция, Bootstrap-CVaR, Black-Litterman, 4-pillar, стресс) в MCP-сервер: тогда «посчитай мой CVaR/риск-вклад/стресс» становится инструментом, доступным Claude Desktop, корпоративным агентам и самому боту. **Готовых MCP-серверов институционального риск-разложения на рынке нет** — данные коммодитизированы, аналитика нет (§6).

### 5.4 🔵 Выбор модели и thinking
- **DEEP-тир** для институционального нарратива можно поднять до **Opus 4.8** (`claude-opus-4-8`, $5/$25, 1M ctx) с **adaptive thinking** (`thinking:{type:"adaptive"}`, `effort:"high"`) — заметный прирост качества рассуждений. ⚠️ На Opus 4.7/4.8 `temperature` и `budget_tokens` **удалены** (вернут 400) — убрать их при миграции (см. migration-гайд).
- **BASE-тир** оставить Haiku 4.5 ($1/$5) — быстро/дёшево.

### 5.5 🔵 Batch API — для офлайн-генерации отчётов
Отчёты складываются по датам (`r/<uid>/<date>/base|deep.html`) — это похоже на расписание. Несрочную генерацию (плановые/ночные прогоны) гнать через **Batch API** = **−50%** к стоимости токенов.

### 5.6 🟣 Claude Agent SDK / Managed Agents — стратегический горизонт
Если продукт двинется в сторону «спроси портфель диалогом / автономный ребаланс-ассистент», **Managed Agents** дают серверный agent-loop, песочницу, MCP-коннектор и **vaults** (хранилище MCP-кредов с авто-refresh — секреты не попадают в песочницу). Сейчас избыточно (бот — это «движок→адаптер→шаблон»), но это естественный путь к agentic-UX.
