# ─────────────────────────────────────────────────────────────────────────────
# Dependency policy (Sprint-2 #5)
# ─────────────────────────────────────────────────────────────────────────────
# Every dependency carries an UPPER BOUND capping it at the validated MAJOR line.
# Rationale: an unbounded `>=` lets `pip install` silently pull the next major of
# numpy / pandas / scikit-learn — where LedoitWolf shrinkage, Ridge fitting or
# pandas dtype/copy semantics can change and SILENTLY alter risk math.  The caps
# below make such a jump a hard, visible failure instead of a hidden regression.
#
# NOTE: exact `==` + `--hash` locking must be generated by CI (`pip-compile
# --generate-hashes`) AGAINST THE DEPLOY IMAGE — not by hand here, where the
# locally-resolved versions differ from production's validated 1.26/2.x line.
# Pinning to a mismatched local resolution would ship untested versions.  Commit
# the produced `requirements.lock` and install it with
# `pip install --require-hashes -r requirements.lock` in the Dockerfile.
# ─────────────────────────────────────────────────────────────────────────────
