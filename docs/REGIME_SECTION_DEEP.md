## 0. TL;DR

Секция «Режим» — детерминированный 2-осевой классификатор фазы рынка (Growth × Cycle) на
моментуме ликвидных US ETF, с ограниченной макро-поправкой из FRED (level ⊕ темп), тремя
слоями подтверждения (FRED-драйверы, детерминированная сверка R3, ИИ-сверка) и квадрант-SVG.
Математика движка **верифицирована на живых числах: confidence 34% реконструируется точно**
(см. §3.4). Основные проблемы секции — не в движке, а в **премиум-визуализации**: перепутанные
подписи Recovery/Slowdown на квадранте, захардкоженные «позитивные» подписи и дата as_of,
сырой токен `diverges` вместо текста ИИ-сверки и «RAG-сигналы», которые на самом деле —
ETF-моментум (§6).

## 2. Математика классификатора (`finance/regime.py`)

### 2.1 Две ортогональные оси

Все входы — уже загруженные MAC3-движком дневные CLOSE-ряды ETF; никаких новых сетевых вызовов.
`_ret(col, win) = P[-1]/P[-win-1] − 1` — простая доходность за окно (60 торговых дней MEDIUM,
120 LONG).

**Growth-ось** («акции лидируют против облигаций?») — среднее списка компонент:

```
growth_components = [ SPY60 − IEF60,          # equity vs treasuries
                      EEM60,                  # аппетит к EM-риску
                      0.5 · SPY120 ]          # 6-мес стабилизатор (полувес)
growth_score = mean(growth_components)
```

**Cycle-ось** («цикличные лидируют против защитных?»):

```
cycle_components = [ XLY60 − XLP60,           # Discretionary vs Staples
                     IWM60 − SPY60 ]          # small vs large (ранний цикл)
cycle_score = mean(cycle_components)
```

Любая недостающая колонка просто выпадает из списка (устойчивость к неполному universe).
Если нет **ни одного** сигнала — `classify` возвращает `None` («режим неизвестен»), как и при
`len(prices) < 65`.

### 2.2 Макро-надбавка (BLOCK 3.4, default-ON, env `REGIME_MACRO_OVERLAY`)

Когда подан FRED-пак, к спискам компонент добавляются **три** поправки **того же масштаба**, что и
ETF-спреды; каждая — бленд **уровня** и **темпа** (rate-of-change), потому что для режима важнее
направление движения серии, чем её абсолют (безработица 4.1% растущая ≠ той же падающей;
инфляционные ожидания 2.3% с разворотом вверх ≠ тем же 2.3% на снижении):

```
GDP (→ growth):      lvl = (GDP_SAAR − 2.0) · 0.020
                     mom = trend₃кв(GDP) · 0.015          # OLS-наклон по ≥4 точкам (≥3 изменений)
                     g_nudge = clip(0.5·lvl + 0.5·mom, ±0.05)

Безработица (→ cycle): lvl = (4.5 − U) · 0.030            # ниже нейтрали = плюс
                     mom = (−trend₃мес(U)) · 0.040        # РОСТ безработицы = минус
                     c_nudge = clip(0.5·lvl + 0.5·mom, ±0.05)

Инфляция (→ growth): lvl = (2.0 − BE₁₀ᵧ) · 0.020          # 10Y breakeven vs таргет ФРС 2%
  (добавлено          mom = (−trend₁мес(BE)) · 0.100      # темп ре-анкоринга: РОСТ ожиданий = минус
   2026-07-04)        i_nudge = clip(0.5·lvl + 0.5·mom, ±0.05)
```

**Логика инфляционной поправки (2026-07-04, по запросу владельца).** Официальные данные по
инфляционным ожиданиям (FRED `T10YIE`, 10Y breakeven — уже в макро-паке, §4.1) анализируются
через **темп изменения** в первую очередь: ожидания, ре-анкорящиеся ВВЕРХ от таргета, означают,
что рынок закладывает ужесточение ФРС → давление на risk-аппетит; дрейф ВНИЗ к таргету —
пространство для смягчения. Уровень (отклонение от якоря 2%) даёт вторую половину бленда.
Поправка входит в **growth-ось** (канал политики/ставки дисконтирования: акции vs трежерис),
а не в cycle: (а) экономически инфляционные ожидания двигают ось «рынок vs облигации», не
ротацию цикличные/защитные; (б) в growth-списке уже 4 компоненты, и третий макро-сигнал там
держит долю макро ≤ 2/5 оси — принцип «TILT, не override» сохранён. Темп по daily-серии
считается за ~1 месяц наблюдений (`_INFL_TREND_LAG=21`), тем же `series_trend` (≥3 изменений).
Сигнал-ключ в `signals`: `macro_inflation_nudge` (+ диагностика `macro_breakeven_trend`).

Ключевые свойства: темп считается `series_trend` — **OLS-наклон по окну**, а не одношаговая
разница (анти-шум, требование «≥3 последовательных изменений», иначе `None` → остаётся только
уровень с полным весом); каждая поправка ограничена `±0.05` ≈ четверть референс-магнитуды —
макро может **наклонить**, но не перехватить ценовой сигнал; статусы серии `ok/stale` принимаются,
`missing/error` — игнорируются (`_usable_macro`); отсутствующая серия просто не входит в ось
(не отравляет). Эскейп: `REGIME_MACRO_OVERLAY=0` возвращает байт-идентичный чисто ценовой
классификатор.

> Примечание к §3: реконструкция confidence 34% ниже сделана по отчёту 2026-07-03, **до**
> добавления инфляционной поправки — в новых отчётах в growth-оси появляется пятая компонента
> `macro_inflation_nudge`, и числа осей/уверенности будут отличаться (это ожидаемое изменение).

### 2.3 Квадрант

```
cycle ≥ 0 · growth ≥ 0 → Expansion    (risk-on, поздний цикл)
cycle ≥ 0 · growth < 0 → Recovery     (risk-on, ранний цикл: цикличные уже лидируют, рынок ещё нет)
cycle < 0 · growth ≥ 0 → Slowdown     (risk-off поздний: рынок растёт, цикличные сдают)
cycle < 0 · growth < 0 → Recession
```

Историческая справка (важно для §6): раньше движок путал Recovery↔Slowdown; **исправлено в
`finance/regime.py`** (комментарий у квадрант-решения) — но, как показано ниже, обе SVG-версии
квадранта до сих пор подписаны по старой (доисправленной) схеме.

### 2.4 Confidence — произведение двух независимых факторов

```
magnitude_factor      = min(1, hypot(growth_score, cycle_score) / 0.20)
directional_agreement = доля компонент (по обоим спискам), чей знак совпадает
                        со знаком назначенной оси
confidence            = magnitude_factor × directional_agreement
```

0.20 — эмпирическая магнитуда «чёткого» режима. Схема убирает старый дефект, когда
`(0.15, 0.04)` раздувался до 100% на одном только отношении магнитуд. Оба фактора публикуются в
`signals` (`confidence_magnitude`, `confidence_directional`) для QC-панели.

## 3. Верификация на живом отчёте (2026-07-03)

Из `window.DEEP.regime`: `name=Expansion, confidence=34, growth=+0.0768, cycle=+0.0243`,
explainers: SPY−IEF **+14.0%**, IWM−SPY **+4.2%**, EEM **+14.7%**; драйверы: GDP 2.10% (темп
−2.70 пп/3кв), U 4.20% (темп −0.09 пп/3м).

### 3.1 Реконструкция cycle-оси — сходится точно

`XLY/XLP` в explainers **нет** → пары Discretionary/Staples в этом universe не было, и
cycle-ось собрана из 2 компонент: `IWM−SPY = +0.042` и `c_nudge`:

```
lvl = (4.5 − 4.2)·0.030 = +0.009;  mom = +0.09·0.040 = +0.0036
c_nudge = 0.5·0.009 + 0.5·0.0036 = +0.0063
cycle_score = (0.042 + 0.0063)/2 = 0.0242 ≈ +0.0243 ✓
```

### 3.2 Реконструкция growth-оси

```
lvl = (2.1 − 2.0)·0.020 = +0.002;  mom = (−2.7)·0.015 = −0.0405
g_nudge = 0.5·(0.002 − 0.0405)… = −0.0193   (замедление ВВП тянет вниз)
growth_score = mean(0.140, 0.147, 0.5·SPY120, −0.0193) = +0.0768
  ⇒ 0.5·SPY120 ≈ +0.039 ⇒ SPY120 ≈ +7.8%  (правдоподобно)
```

### 3.3 Квадрант и точка

`growth>0, cycle>0` → **Expansion** ✓; точка на квадранте `dot={growth:0.0768, cycle:0.0243}` —
верхне-правый квадрант, близко к центру (слабый сигнал — виден глазом).

### 3.4 Confidence 34% — точная сверка

```
magnitude        = hypot(0.0768, 0.0243) = 0.0806
magnitude_factor = 0.0806/0.20 = 0.403
знаки компонент: growth {+, +, +, −(g_nudge)} → 3/4;  cycle {+, +} → 2/2
directional_agreement = 5/6 = 0.833
confidence = 0.403 × 0.833 = 0.336 → 34% ✓  (в CoVe: «Expansion · confidence 34%»)
```

То есть низкая уверенность — **честный сигнал**: вектор всего в 40% от референс-магнитуды и
одна компонента (замедляющийся ВВП) голосует против квадранта. Ровно это и говорит ИИ-комментарий
(«уверенность низкая (34%): темп ВВП замедляется (−2.7)») — LLM согласован с числами движка.

## 4. Слои подтверждения вокруг ярлыка

### 4.1 FRED-драйверы (панель «Сигналы-драйверы»)

`MacroFeed.get_regime_drivers()` — 6 серий: `T10Y2Y` (наклон кривой), `BAMLH0A0HYM2` (HY OAS),
`VIXCLS`, `T10YIE` (breakeven), `UNRATE`, `A191RL1Q225SBEA` (Real GDP SAAR). QualityGate на
свежесть (окна по каденции: daily 5 дн; UNRATE 75 дн; GDP 210 дн — фикс §−14 B-3 отсчёта от даты
наблюдения) и sanity-диапазон. `_build_macro_drivers_panel` форматирует значение по юниту
(HY → bp ×100; pp со знаком; VIX — индекс) и добавляет **чип темпа** `_macro_series_trend`:
OLS-Δ по истории `history_30d` с lookback по каденции (daily→21 обс ≈1м, monthly→3 ≈3м,
quarterly→3 ≈3кв) → «▲ +0.97 bp за 1м» / «▼ −2.70 пп за 3кв».
Важно: **3 дневные серии (кривая 10Y−2Y, HY OAS, VIX) — контекст/подтверждение, в классификацию
не входят**; в осях участвуют GDP, UNRATE и — с 2026-07-04 — breakeven-инфляция (T10YIE)
через overlay (§2.2).

### 4.2 Детерминированная сверка R3 (`regime_consistency`, без LLM)

`risk_on = label ∈ {Expansion, Recovery}`; стресс-сигналы: инверсия `10Y−2Y < 0`, широкий HY
`> 5.5%` (550 бп), страх `VIX > 25`. Risk-on при ≥2 стресс-сигналах → `diverges`;
risk-off при спокойном макро → `diverges` (возможен ранний разворот); иначе `aligned`.
Live: +0.35 пп / 274 бп / 16.6 → `aligned`. **В Премиум-отчёте этот блок не отображается**
(см. R-8 в §6) — его рендерит только классический v3.

### 4.3 ИИ-сверка (`regime_confirmation`, DEEP-only, advisory)

Opus получает драйверы/беты/банковский контекст и обязан вернуть структуру
`{stance: confirms|partial|diverges, summary, signals[≤6]}`; каждая строка проходит
`_strip_unverified_rag_citations` (цитата банка не может ссылаться на неизвлечённый отчёт).
Live: `stance=diverges`, 6 сигналов (2 контр: «ВВП замедляется», «беты высоки при conf 34%»).

### 4.4 RAG-подтверждение (`regime_rag_confirm`)

`tg_bot._fetch_rag_context` делает отдельный запрос в ChromaDB
(`Market regime {label} GDP growth recession…`) и берёт до 3 строк ≥30 символов. Сейчас
**всегда пусто** — база RAG не наполнена (см. `docs/BANK_RAG_COVE_DIAGNOSIS.md`); кроме того,
ключ payload'а нигде не рендерится (dead key, R-6 в §6).

## 5. Дизайн и визуализация (Премиум, секция 04 «Стресс · Режим»)

Раскладка `RegimeBlock` (`deep-stress-regime.jsx:40-131`) — grid 12 колонок:

1. **Карточка квадранта** (5/12, ≥lg): заголовок «Growth × Cycle / Координаты режима», чип
   `60-day`, по центру `RegimeQuadrant` (SVG 300×300).
   Геометрия чарта (`deep-charts.jsx:303`): `pad=34`, внутренняя область `inner=232`,
   полудиапазон осей `SCALE=0.22`; проекции `X(v)=cx+v/0.22·116`, `Y(v)=cy−v/0.22·116`
   (X — cycle, Y — growth, вверх = плюс). Точка «сейчас»: золотое ядро r=7 + ореол r=13 +
   пунктирный лидер из центра + подпись `G +0.08 · C +0.02`. Квадранты залиты фирменными
   цветами (Expansion — золото 14% непрозрачности как «целевая» фаза, остальные 5%).
2. **Карточка сводки** (7/12, 2026-07-05 — всё data-driven): ярлык 32px light
   (`Expansion · Экспансия`, `label_ru` из payload), строка «Уверенность модели **N%** ·
   K подтверждающих сигнала» (K = только ✓-буллеты); чип фазы по режиму (Expansion→«Рост»
   сейдж / Recovery→«Восстановление» сейдж / Slowdown→«Замедление» золото /
   Recession→«Спад» ржавый); мини-карточки Growth/Cycle со знаковым форматом `%+.2f`,
   цветом по знаку и подписью по знаку («здоровый рост»/«рост под давлением»,
   «цикл. экспансия»/«цикл. сжатие»); заголовок драйверов несёт живой
   `as_of {driversAsOf}`; пилюля статуса серии — RU («актуально»/«устарело»/…).
3. **Полоса ИИ-сверки** (12/12, градиент сейдж): заголовок красится по `confirmStance`
   (confirms → «ИИ подтверждает режим» ✓ сейдж · partial → «Частичное подтверждение ИИ»
   ⚠ золото · diverges → «ИИ расходится с моделью» ⚠ ржавый), абзац `r.confirm` —
   человекочитаемый `summary` (не сырой токен); 2-колоночный список `confirmBullets`
   (✓ сейдж / ⚠ золото по `ok`); строка детерминированной сверки R3 (`consistency`:
   «✓/⚠ …» сейдж/золото); внизу — чипы `{RAG|Моментум} · {signal}` (метка «RAG» только
   когда чипы реально из `regime_rag_confirm`, флаг `ragBacked`).
4. **AI-комментарий к режиму** (12/12, glass-карта): `regimeAI` — свободный текст Opus.

Классический v3 (fallback, `report_deep_v3.html:2955+`) рисует тот же квадрант серверным SVG:

## 8. Как читать секцию (для пользователя отчёта)

- **Ярлык + точка**: фаза рынка по цене; чем дальше точка от центра — тем чётче режим.
- **Уверенность <50%** — режим у границы: сейчас 34% = «Expansion, но слабый» (замедление ВВП
  голосует против — видно и в контр-сигнале ИИ, и в отрицательной GDP-надбавке).
- **Драйверы** — независимый макро-контекст (уровень + темп); кривая/HY/VIX в классификацию
  не входят (их дело — подтвердить или поставить под сомнение ценовой моментум), а GDP,
  безработица и breakeven-инфляция дают ограниченные ±0.05 надбавки осям (§2.2).
- **ИИ-сверка** — advisory-«второе мнение», не рекомендация; при `diverges` доверяйте цифрам
  движка и контр-сигналам больше, чем ярлыку.
