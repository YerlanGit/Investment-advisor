## Где движок СТРОЖЕ фреймворка (осознанные отклонения — фиксируем)

1. **Vol:** фреймворк использует gross Σwᵢσᵢ (сам оговаривает «true vol will be lower»). Мы
   считаем **истинную портфельную σ через ковариацию** (EWMA⊕Ledoit-Wolf уже есть) для КАЖДОГО
   из 5 сценариев — это и есть главная добавленная ценность тира vs ручной таблицы. Gross-число
   можно показать серым как reference.
2. **Sharpe:** фреймворк = Return/GrossVol без RFR. Мы: (Return − RFR)/σ_p — конвенция движка
   (Base Currency + RFR 4.50%), иначе противоречим остальному отчёту. В таблице подписать
   «Sharpe (RFR-adj)».
3. **3Y Ann. Return:** требует ≥756 торговых дней; текущее окно 730 кал. (~500 торг.) не
   покрывает. **Зависимость: включение `HISTORY_LOOKBACK_DAYS=1825` (Data Resilience §4-б) —
   пререквизит Панели A с 3Y-метриками**; до этого честно показываем 2Y с подписью окна.
4. **Survivorship/look-ahead (Панель B):** сетка сигналов по `df[df.index <= t]`, тестируются
   только живые тикеры — предупреждение в шаблоне обязательно.

## Worked example = приёмочный тест

Пользовательский кейс (12 позиций базы; SELL AAPL 15%+MSFT 15% → BUY NVDA 10%/TSM 10%/
XOM 5%/GLD 5%; SoftBank отсутствует в базе → no-op) кладётся фикстурой
`tests/fixtures/scenario_worked_example.json`. Тест проверяет **направления**, не байты
(наши vol/Sharpe точнее gross-чисел примера): Sharpe↑, E[r]↑, β↑ (~0.6→~0.7), P/E↓,
инфляционный хедж 10→15%, «SoftBank: позиции нет — продавать нечего» (framework-правило:
sell-инструкция по отсутствующей позиции = валидный no-op с пометкой).

## Жёсткие ограничения (в шаблон тира, дословно)

1. Прошлое ≠ будущее; Панель B — иллюстрация правила, не гарантия.
2. Survivorship bias: тестируются только сегодняшние тикеры — результат смещён вверх.
3. Панель A — ожидания на параметрах движка, не прогноз; сценарии не являются ИИР.
4. Комиссии/проскальзывание в v1 не учтены (v2: константа 10 б.п./сделка).
