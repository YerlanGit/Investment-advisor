| Символ | Смысл |

## 1. Подготовка данных

### 1.1 Загрузка и чистка — `investment_logic::MAC3RiskEngine.get_market_data`, `::math_firewall`
- Окно запроса: `HISTORY_LOOKBACK_DAYS` (default **1825 календарных** ≈ 1260
  торговых ≈ 5 лет); параметр `days` в `history.py` календарный
  (`today − timedelta`).
- `math_firewall`: `±Inf → NaN`, затем **только ffill** (внутренние гэпы).
  Ведущие NaN (до листинга) СОХРАНЯЮТСЯ (F-6) — bfill создавал фиктивные
  нулевые доходности, занижая σ/β молодых бумаг (look-ahead bias).
- **Зачем:** одна битая котировка не должна отравить ковариацию; история
  моложе листинга не должна существовать.

### 1.2 Валютный слой H2/H3 — `currency.py`
- Каждая колонка цен переводится в валюту отчёта поточечным умножением на
  FX-серию с **лагом T−1** (`align_fx_to_prices(lag_one_day=True)`) — FX-вола
  естественно попадает в ковариацию, синтетический FX-фактор не нужен.
- LSE-листинги котируются в пенсах (GBX): цена **÷100 → GBP** → кросс-курс
  (F-7; иначе ×100 ошибка оценки).
- RFR подбирается ПОД валюту отчёта (`get_rfr_for_currency`) и переводится в
  дневную **геометрически** (H3):

  ```
  rf_daily = (1 + rf_annual)^(1/252) − 1        currency::daily_rfr_geometric
  ```
  Линейное rf/252 завышает дневную ставку на ~r²/(2·252) и смещает Sortino.
- **Композитный риск-гейдж (0–100, 2026-07-18):** базовые 3 сигнала — vol/0.40, |CVaR|/cvar_base(мандат), maxERC/50 — с мандатными весами (`composite_risk_score`). Плюс **bounded structural/tail аггравторы** (opt-in, только повышают): MaxDD |DD| 20→50% ⇒ +0…15, доля топ-сектора 50→75% ⇒ +0…10, плечо (long/net) 1.0→1.5x ⇒ +0…6. `analyze_all` переопределяет гейдж после расчёта секторов/плеча; `simulate_after_plan` получает те же мандат+аггравторы, поэтому verdict-гейдж и «до/после» согласованы. Отсутствие сигнала (None) ⇒ вклад 0 (обратная совместимость).
- **Зачем:** Sharpe «USD-активы против KZT-ставки» — утечка кросс-валютной
  премии; валюта цен и валюта RFR обязаны совпадать.

## 2. Факторная модель MAC3 (Barra-стиль) — `investment_logic::calculate_structural_risk`

### 2.1 Факторы
10 ETF-прокси: Market=SPY, Momentum=MTUM, Value=VLUE, Quality=QUAL, Size=IWM,
Volatility=SPLV, Commodities=DBC, Rates=IEF, EM_Equity=EEM, EM_Bond=EMB
(`factor_tickers`).

### 2.2 Иерархическая ортогонализация (default-ON) — `::orthogonalize_factors_hierarchical`
Стилевые/EM-факторы несут ту же рыночную бету, что и ядро (κ≈62,
max|corr|≈0.89) → каждый «ребёнок» residual-изуется OLS против родителя
(Momentum/Value/Quality/Size/Volatility/EM_Equity → Market; EM_Bond → Rates):

```
child_resid(t) = child(t) − Σ_p β̂_{child→p}·parent_p(t) + mean(child)
```

Имя и среднее фактора сохраняются; удаляется только дублируемая рыночная
компонента. β̂_{child→parent} экспортируются (F-1) — стресс-движок обязан
преобразовать сырые шоки в это же residual-пространство (§6.2).
Откат: `FACTOR_ORTHOGONALIZE=0`.

### 2.3 Гейты данных (порядок применения)
| Гейт | Порог | Что делает | Где |
|---|---|---|---|
| sparse-guard (F-6) | история актива < **60** торг. дн (`MIN_OVERLAP_TDAYS`) | имя ИСКЛЮЧАЕТСЯ из структурной модели (вес честно разбавляет риск; Action Plan аннотирует «вне модели», F-20) | `calculate_structural_risk` |
| min_obs регрессии (F-3) | общее окно < max(2K, 30) | структурная модель пропускается целиком (беты на <2K точек — интерполяция шума) | там же |
| гейт форвардной панели (F-14) | окно регрессии < **252** | портфельная «Ожид. дох./фвд-Sharpe» → «—» | `analyze_all` |

Окно регрессии = пересечение дат выживших колонок (row-dropna лог-доходностей).

### 2.4 Беты — Ridge-регрессия
На каждый актив: `y = дневные лог-доходности актива`, `X = f_data` (K факторов):

```
model = Ridge(alpha=0.001, fit_intercept=True)     β = coef, α̂ = intercept
```

alpha=0.001 — стабилизация коррелирующих факторов БЕЗ экономически заметного
сжатия (alpha=1.0 сжимал бы дневные беты на ~98%).

**Специфический риск:** `spec_var = Var(residuals, ddof=1)`;
`Residual_Vol_Ann = std(resid, ddof=1)·√252`.

### 2.5 SE(β) — P-4 (аудит B1)
```
SE(β_k) = σ_ε / (σ_{f_k} · √T)
```
σ_ε — std остатков (ddof=1), σ_{f_k} — дневная std фактора k, T — окно
регрессии. Формула точна для ОРТОГОНАЛЬНЫХ регрессоров (факторы
ортогонализованы по умолчанию; в legacy raw-режиме это нижняя граница).
→ `portfolio_metrics.beta_standard_errors` {актив: {фактор: SE}};
вердикт `beta_reliability = "noisy_short_window"` при max SE(β_Market)/|β| > 0.5
на окне < 252 → ⚠-чип «Надёжность бет» в отчёте.
**Зачем:** β=1.8 на 80 днях статистически неотличима от 1.0 — без SE советник
принимает шум за сигнал.

### 2.6 Vasicek-сжатие бет — P-4b, за флагом `BETA_SHRINKAGE` (default OFF)
```
w = T/(T + n₀),  n₀ = BETA_SHRINKAGE_PRIOR_OBS (120)
β* = w·β̂ + (1−w)·prior         prior: Market → 1.0 (для реестровых LETF → L),
                                        стили/EM → 0.0
```
Интерсепт и остатки ПЕРЕсчитываются с β*, чтобы D осталась согласованной.
Выключено по умолчанию: меняет Σ на ступенчатых книгах → требует отдельной
валидации (METHODOLOGY §2.2/§5).

### 2.7 Ковариация факторов F
```
F = 0.7·EWMA_cov(halflife=63) + 0.3·LedoitWolf_cov          (дневная)
```
- EWMA hl=63 торг. дн (~3 мес): дневной decay λ = 0.5^(1/63) ≈ 0.989 —
  СОЗНАТЕЛЬНО плавнее RiskMetrics λ=0.94 (≈hl 11 дн) ради стабильности весов.
- Ledoit-Wolf — сжатие при малом N (это сжатие КОВАРИАЦИИ ФАКТОРОВ, не бет).
- Бленд не гарантированно PSD → проекция Хайэма `_nearest_psd` (клип
  отрицательных собственных значений) — иначе возможно w'Σw < 0.

### 2.8 Диагностика мультиколлинеарности — BLOCK 4.6
На корреляционной матрице F: `max|off-diag corr|` и число обусловленности κ;
`near_collinear = (max|corr| > 0.95 или κ > 30)` → warning + CoVe-строка.
**P-5:** `max_corr_ci95` — Fisher-ДИ (§8.2) на max|corr| при n = окно
регрессии; `corr_window_days` — само окно. На коротких окнах ДИ показывает,
что знак/величина ρ̂ не определены.

### 2.9 Структурная ковариация активов
```
Σ_daily = B·F·Bᵀ + D            Σ_ann = Σ_daily · 252
```
**Что даёт:** ковариация через ОБЩИЕ факторы (систематика) + независимая
идиосинкратика. Off-diagonal F учитывает коррелирующие факторы ровно один
раз — двойного счёта нет.

## 3. Портфельный риск (реализованная панель)

### 3.1 Волатильность и Euler-декомпозиция
```
σ_p = √(wᵀ·Σ_ann·w)                                (кламп w'Σw ≥ 0)
MCTR_i = (Σ_ann·w)_i / σ_p                          маржинальный вклад
ERC%_i = w_i·MCTR_i / σ_p · 100                     Σ ERC% = 100% (тождество Эйлера)
```
**Зачем:** ERC% отвечает «какую долю ТЕКУЩЕГО риска несёт позиция i» —
основа hotspot-флага (ERC > 20% → принудительный Trim, §10.4) и
концентрационной компоненты риск-скора.

### 3.2 Базис реализованных метрик — маскированный композит (F-15/F-22)
Серия портфеля строится по ПОЛНОЙ панели с по-дневной ренормализацией весов
реально торговавшихся имён (конвенция композитных индексов MSCI/GIPS):

```
r_port(t) = Σᵢ wᵢ·rᵢ(t)·1[торговался] / Σᵢ wᵢ·1[торговался] · Σw
```
- день входит при покрытии ≥ 50% весов (`MIN_DAILY_COVERAGE`,
  `period_returns::build_portfolio_log_returns`);
- множитель ×Σw сохраняет кэш-разбавление;
- на полных панелях путь бит-в-бит равен легаси `a_data @ w` (композит
  подключается ТОЛЬКО когда удлиняет ряд);
- окно регрессии (§2.3) НЕ меняется — от композита выигрывает только
  реализованная панель. Телеметрия: `realized_window_days`.

**Зачем:** одно молодое имя не должно срезать 5-летнюю историю книги до
своего возраста (Sharpe 12М был невычислим, спарклайны пустели).

### 3.3 Доходность, Sharpe, Sortino
```
AnnReturn = exp(mean(r_port)·252) − 1               геометрическая (Itô: арифм.
                                                     mean·252 завышает на ~σ²/2)
Sharpe  = (AnnReturn − rf_annual) / σ_p
Sortino = (AnnReturn − rf_annual) / DownsideVol
DownsideVol = √(mean(min(r_port − rf_daily, 0)²))·√252     (Satchell: знаменатель
                                                            по ВСЕМ N, MAR=0)
```
**Смешанный базис Sharpe (F-4, осознанно):** числитель — реализованная
геометрическая доходность за ВСЁ окно; знаменатель — СТРУКТУРНАЯ σ
(EWMA hl=63 ⊕ LW, взвешена к последним ~3 мес). В спокойном свежем рынке
оценка выше классического sample-Sharpe, после стресса — ниже. Базис
раскрыт в отчёте (`sharpe_basis_note` → Integrity-панель).

### 3.4 VaR / CVaR (исторические, 1-дневные)
```
VaR_95  = перцентиль(r_port, 5%)
CVaR_95 = mean(r_port | r_port ≤ VaR_95)            хвостовое среднее
```
Параметрический (гауссов) VaR в движке НЕ используется — хвост не
недооценивается предположением нормальности.

**Bootstrap-ДИ CVaR** — `::_bootstrap_cvar` (stationary block bootstrap,
Politis–Romano 1994): блоки геометрической длины (средняя ≈ √N) сохраняют
автокорреляцию дневных доходностей; 2000 репликаций → `point/lo95/hi95`.
Сид детерминирован данными (SHA-256 от округлённого окна) — «same input →
same CI», но CI движется при сдвиге окна (H5). Требует ≥ 60 наблюдений,
иначе точечная оценка с чипом «мало данных».

**Надёжность (P-3):** `var_reliability = "insufficient_history"` при
`realized_window_days < 100` (`MIN_RELIABLE_TAIL_OBS`) → ⚠-чип. Для
стабильного 95%-хвоста нужно ~100 точек.

**χ²-ДИ σ (P-5):** `volatility_ci = {lo_mult, hi_mult}` по §8.1 на
T = realized_window_days → чип «σ ×[0.86, 1.20]» и т.п.

### 3.5 Max Drawdown
```
equity(t) = exp(Σ_{τ≤t} r_port(τ));   MaxDD = min(equity/cummax(equity) − 1)
```
Пик-к-впадине по ФАКТИЧЕСКОЙ кривой — для daily-reset ETP эрозия
компаундинга входит автоматически (цены ETP уже содержат ресет; тест:
500-дневный путь 2× даёт MaxDD −97%, «наивные 2×» дали бы −44%).

### 3.6 Marginal VaR — `::_marginal_var`
```
dVaR/dw_i ≈ [VaR(w⁺ᵢ) − VaR(w⁻ᵢ)] / (2h),  h = 0.005
```
Бамп ±h **бюджетно-нейтрален**: остальная книга масштабируется, Σw
неизменна (финансируем покупку продажей остальных pro-rata) — иначе
измерялась бы чувствительность С ПЛЕЧОМ (F-1-fix). Требует ≥60 наблюдений.
**Зачем:** дополняет ERC: «на сколько сдвинется 1-дневный VaR при +1% веса».

### 3.7 ATR — `::calculate_atr`
```
TR = max(H−L, |H−C_prev|, |L−C_prev|);   ATR = RMA_Wilder(TR, 14)  (α = 1/14)
```
Фолбэк без OHLC: RMA(|ΔClose|). НЕ pandas `span` (α=2/(n+1) — это EMA,
пересглаживало бы). Питает стоп/тейк-уровни Action Plan.

## 4. Форвардные оценки (ex-ante)

### 4.1 Ожидаемая доходность актива — β·μ (F-14)
```
μ_f (daily log) = mean дневных лог-доходностей фактор-ETF (ПОЛНАЯ история)
E[r_i]_daily    = Σ_k β_ik·μ_k                       (α̂ ИСКЛЮЧЁН из форварда)
E[r_i]_annual   = exp(E[r_i]_daily·252) − 1          кламп в [−50%, +100%]
```
α̂ (реализованный идиосинкразический дрейф) — не персистентная премия;
его аннуализация на коротком окне дала живой баг «218.4% · Sharpe 11.29».
Кламп — belt-and-suspenders против шумной β (счётчик
`expected_return_clamped_n` в QC).

### 4.2 Плечевые daily-reset ETP — P-1/F-23 (`leveraged.py`, `::apply_leveraged_forward`)
Дневное лог-приближение k×-ресета: `r_lev ≈ L·r_u − ½L(L−1)σ_u²` — второй
член (variance drag) КОНТРАКТНЫЙ (ежедневная ребалансировка), не шум.

Маршрутизация по реестру `LEVERAGED_ETP_REGISTRY` {base: (L, underlying, ER)}:

```
L известен и σ_ETP доступна (≥60 obs):        МЕТОД "contractual"
    σ_u   = σ_ETP / |L|
    drag  = −½·L·(L−1)·σ_u²        (≤0; для L∈(0,1] срезается в 0)
    fee   = −ER/252                (0 при ER=None)
    E[r]_daily += drag + fee       (α̂ игнорируется — без двойного счёта)

L неизвестен (имя в реестре без параметров):  МЕТОД "empirical_alpha" (F-23)
    E[r]_daily += min(α̂, 0)       (drag из данных; положительная α никогда
                                    не добавляется — anti-218%-guard)
```
Общий вид работает и для inverse: L<0 ⇒ L(L−1)>0 ⇒ drag<0 (инверсники
деградируют так же). QC: `leveraged_adjustments` {тикер: {method, L,
underlying, drag_daily, fee_daily, drag_ann_pct, fee_ann_pct}} → чип
«Плечевые ETP» (drag и комиссия РАЗДЕЛЬНО — C5). Расширение: env
`LEVERAGED_ETP_PARAMS="TICK:L[:UNDERLYING[:ER]];…"` / `LEVERAGED_ETP_EXTRA`.

**Тождество Sharpe под плечом** (закреплено тестом):
`Sharpe_LETF ≈ Sharpe_u − ½(L−1)·σ·√T`; кейс L=2, σ=1.35, μ=0.15 →
0.11 − 0.675 ≈ **−0.57** (не «удвоенный» +0.11).

### 4.3 Портфельный форвард — BLOCK 5
```
E[r_port] = Σᵢ wᵢ·E[rᵢ] + w_cash·rf                (кэш-drag в цене)
Expected_Sharpe = (E[r_port] − rf) / σ_p
```
Печатается ТОЛЬКО при окне регрессии ≥ 252 (F-14), иначе «—»
(`expected_return_window_days` в QC).

## 6. Стресс-движок — `stress.py`

### 6.1 Линейный β·shock (базовый метод)
Сценарий = вектор периодных шоков по факторам (напр. Tech sell-off:
Market −10%, Momentum −15%, Quality −5%; калибровки — по историческим
аналогам, задокументированы в каталоге `DEFAULT_SCENARIOS`):

```
Δᵢ = Σ_f β_{i,f}·shock_f          ΔPnL_port = Σᵢ wᵢ·Δᵢ = wᵀ·B·shock
```
Стандартный quick-stress (Goldman MarketRisk / Aladdin) — линейное
приближение, валидное для шоков одноразрядных процентов.

### 6.2 Residualize shocks (F-1) — `::residualize_shocks`
Каталог откалиброван в RAW-движениях факторов, а беты живут в
residual-пространстве (§2.2) → сырой шок ребёнка корректируется:

```
shock_resid_child = shock_raw_child − Σ_p β̂_{child→p}·shock_raw_p
```
(родительские шоки не меняются; ребёнок БЕЗ шока в сценарии тоже
трансформируется: 0 − Σβ̂·shock_parent). Итог: `wᵀB_resid·shock_resid ≡
wᵀB_raw·shock_raw` — стресс-таблица ИНВАРИАНТНА к FACTOR_ORTHOGONALIZE.

### 6.3 Convex-cap (H4) — `::_convex_cap`
Для ОБЫЧНЫХ имён |Δᵢ| > 20% сглаживается C¹-гладкой сатурацией с
асимптотой ±35%:

```
x' = sign(x)·(T + (C−T)·(1 − e^{−(|x|−T)/(C−T)}))   при |x| > T=0.20, C=0.35
```
**Зачем:** беты оценены в нормальном режиме и нелинейно декомпрессируются в
хвостах — линейная экстраполяция −50% для β=2-имени бессмысленна.

### 6.4 Path-dependent LETF (P-7)
Для реестровых имён с известным L (σ_ETP передаётся из analyze_all) cap НЕ
применяется — выпуклость КОНТРАКТНАЯ:

```
X_u = Δᵢ_linear / L                (беты ETP уже несут ~L× базу)
Δᵢ  = (1+X_u)^L·exp(−½L(L−1)σ_u²·63) − 1          горизонт = квартал 63 дн
```
2× при базе −25%: линейно −50% → cap −33%, path-dependent **−46.5%** —
честный хвост. Рост тоже корректен: (1.1)²·drag − 1 ≈ +15%, не +20%.
Флаги: `by_asset[].path_dependent`, счётчик `letf_path_n`.

### 6.5 Внутрипериодная просадка и восстановление
```
est_dd = port_pct − σ_p·√0.25                (только для убыточных сценариев:
                                              худший день ~1σ_квартал глубже закрытия)
recovery_months = ln(1/(1−|est_dd|)) / ln(1 + r_monthly)
r_monthly = (1 + rate)^(1/12) − 1,  rate = clamp(AnnReturn, 8%, 18%)
```
Восстановление — геометрическое, по СОБСТВЕННОМУ потенциалу книги,
но не быстрее клампа (несустейнебл CAGR не фабрикует быстрый отскок).

Дисклеймер `STRESS_TEST_DISCLAIMER` присутствует всегда.

## 7. Бенчмарки, периоды, серии — `period_returns.py`, `portfolio_series.py`

### 7.1 Tracking Error / Information Ratio — `::compute_benchmark_stats`
PER-PAIR inner-join портфельной и бенчмарк-серии (короткий бенчмарк режет
только СВОЮ строку):

```
diff(t) = r_port(t) − r_bm(t)                     (лог-пространство)
TE      = std(diff, ddof=1)·√252
IR      = mean(diff)·252 / TE                     (числитель и знаменатель —
                                                   ОДНО окно, одна шкала)
```
Дисплейные годовые — геометрические `exp(mean·252)−1`; в IR не входят.

### 7.2 Мультипериодные доходности — `::compute_period_returns_table`
Окна в ТОРГОВЫХ днях: 1м=21, 3м=63, 6м=126, 12м=252 + календарный YTD.
`r_cum = exp(Σ r_log) − 1` по хвосту выровненного окна; NaN внутри окна
отбрасываются, окно целиком из NaN → None (не фиктивный 0).

### 7.3 KPI-спарклайны — `portfolio_series::compute_kpi_trend_series`
Источник — готовый композит `port_log_returns` (F-21). 12 снапшотов по
последним 252 дням; в каждом — CVaR/Sharpe/MaxDD на трейлинг-окне 60 дн
(теми же формулами §3). Легаси-фолбэк фильтрует колонки < 60 дн.

## 8. Статистический inference малых окон — `finance/inference.py` (P-5)

### 8.1 χ²-ДИ волатильности — `::sigma_ci_multiplier`
```
σ_true ∈ [ s·√((T−1)/χ²_{0.975,df}) ,  s·√((T−1)/χ²_{0.025,df}) ],  df = T−1
```
χ²-квантили — приближение Wilson–Hilferty (без scipy):
`χ²_p ≈ df·(1 − 2/(9df) + z_p·√(2/(9df)))³`, z_p из
`statistics.NormalDist.inv_cdf`. Эталоны: T=20 → ×[0.76, 1.46];
T=31 → ×[0.80, 1.34]; T=1250 → ×[0.96, 1.04].
**Куда идёт:** `volatility_ci` → чип «Окно риск-метрик».

### 8.2 Fisher z-ДИ корреляции — `::fisher_rho_ci`
```
z = arctanh(ρ̂);  SE(z) = 1/√(n−3);  ДИ_ρ = tanh(z ± z_crit·SE)
```
Эталон: ρ̂=0.30, n=20 → [−0.16, 0.66] — интервал ПЕРЕСЕКАЕТ ноль: знак
корреляции на таком окне статистически не определён, и код не имеет права
утверждать значимость. **Куда идёт:** `factor_diagnostics.max_corr_ci95`.

## 9. Композитный риск-скор 0–100 — `scoring::composite_risk_score` (SSOT)

```
score = w_vol·min(σ_p/0.40, 1)·100 + w_cvar·min(|CVaR|/base, 1)·100
      + w_erc·min(maxERC/50, 1)·100
```
Мандатная калибровка (H4, `_RISK_MANDATE_MATRIX`):

| Мандат | w_cvar | w_vol | w_erc | cvar_base |
|---|---|---|---|---|
| CONSERVATIVE | 0.60 | 0.30 | 0.10 | 0.03 |
| MODERATE | 0.40 | 0.40 | 0.20 | 0.065 |
| AGGRESSIVE | 0.20 | 0.50 | 0.30 | 0.08 |

Консерватор хвосто-чувствительнее (выше вес CVaR И жёстче база — тот же

## 10. 4-Pillar скоринг — `scoring.py` + `scoring_orchestrator.py`

### 10.1 Robust z (общая утилита) — `scoring::robust_z`
```
z = (x − median(ref)) / (1.4826·MAD(ref)),  клип ±3
```
None при <5 конечных точек рефа или MAD=0. Медиана+MAD вместо mean/std —
устойчивость к выбросам одного филинга. Каскад рефов для F-пиллара
(`_sector_z`): (1) внутрипортфельная секторная когорта ≥5 имён → (2) живая
SEC-когорта лидеров сектора → (3) статические бенчмарки 2020-25 → (4) None
(«нет данных», не фабрикация).

### 10.2 Пиллары (все клипятся)
**F — Fundamentals (−2…+2)** — `fundamentals_score`: ROE_z ±1, OpMargin_z ±1,
D/A_z (инверсия: < −0.5 → +0.5, > +1 → −1 — плечо карается жёстче),
RevGrowth_z ±1, FCF-margin_z ±1, macro-alignment ±0.5 (согласие сектора с
режимом §12, масштаб = 0.5·confidence), fundamental-momentum ±0.5
(YoY-тренд маржи ∈[−1,1] из SEC ×0.5).

**V — Valuations (−2…+2)** — `valuations_score`: P/E_z, P/B_z (дешёво
z<−1 → +1, дорого z>+1 → −1) + FCF-yield_z с ОБРАТНЫМ знаком (высокий
yield = дёшево → +1). P/E только при NI>0, P/B при equity>0
(`_compute_valuation_ratios`) — отрицательная прибыль НЕ «маленький P/E»,
а отсутствие метрики (E2 аудита).

**T — Technicals (−2…+2)** — сумма §11, клип.

**C — Credit (−2…+1, асимметричный)** — `credit_score`: CDS <40 бп → +1,
90–150 → −1, ≥150 или Δ7д>+20% → −2; Altman Safe/Distress ±1; Piotroski
≥7 → +0.5 / ≤3 → −0.5; InterestCoverage >5× → +0.5 / <1.5× → −1.

**Guard применимости (P-6):** commodities/sovereign-ETF И **LETF-обёртки**
(реестр §5) → `fundamentals_applicable=False`, `credit_applicable=False` —
у свопной обёртки нет отчётности, F/C рендерятся «—» и выходят из
знаменателя видимого Total (иначе фантомный макро-тилт).

### 10.3 Total → действие — `action_from_total`
```
Total = clip(F+V+T+C, −6, +6)
≥+3 Strong Buy · ≥+1 Buy · >−1 Hold · >−2 Trim · иначе Sell
```

### 10.4 Hotspot-override
`ERC%ᵢ > 20` (`HOTSPOT_TRC_PCT`, SSOT) и Total > −1 → принудительный
**Trim** — концентрация бьёт скор. Тот же порог — правило GK-1 гейткипера.

## 12. Режимный классификатор — `regime.py`

Две ортогональные оси из 60/120-дневных доходностей уже загруженных ETF:

```
growth = mean[ (SPY₆₀ − IEF₆₀), EEM₆₀, 0.5·SPY₁₂₀ ]        акции vs облигации
cycle  = mean[ (XLY₆₀ − XLP₆₀), (IWM₆₀ − SPY₆₀) ]           циклики vs защитные
```

Квадранты: growth≥0·cycle≥0 → **Expansion**; growth<0·cycle≥0 →
**Recovery**; growth≥0·cycle<0 → **Slowdown**; иначе **Recession**.

```
confidence = min(1, √(g²+c²)/0.20) × (доля компонент, согласных с квадрантом)
```
(магнитуда × направленное согласие; обе компоненты раскрыты в QC).

**FRED-оверлей (BLOCK 3.4, default-ON):** GDP (level vs 2% ⊕ ускорение),
безработица (level vs 4.5% инверт. ⊕ −тренд), 10Y-breakeven (level vs 2%
инверт. ⊕ −тренд → growth-ось через policy-канал). Каждый сигнал =
50/50 level⊕trend, тренд — OLS-наклон по ≥4 точкам (≥3 изменений,
`series_trend`), клип **±0.05** — макро ТИЛТУЕТ, но не перехватывает цену.
Выход `REGIME_FAVOURED/PENALISED_SECTORS` питает macro-alignment F-пиллара.

## 13. Black-Litterman — `black_litterman.py`

```
π    = δ·Σ·w                                        равновесный prior (реверс-
                                                     оптимизация текущих весов)
Ω    = diag(P·τΣ·Pᵀ)·(1−c)/c                        Idzorek: уверенность c∈(0,1]
μ_BL = π + τΣ·Pᵀ·(P·τΣ·Pᵀ + Ω)⁻¹·(Q − P·π)          He–Litterman closed form:
                                                     решается ТОЛЬКО k×k система
w*   = (δΣ)⁻¹·μ_BL   → клип [0,1] → нормировка Σ=1
```
τ=0.05 (канон), δ — из мандата (CONSERVATIVE 4.0 / MODERATE 2.5 /
AGGRESSIVE 2.0, `_MANDATE_BL_CONSTRAINTS` в investment_logic).

**Констрейнты (в порядке применения):**
1. turnover-бленд: если active share = Σ|w*−w|/2 > cap (0.15/0.25/0.35 по
   мандату) → w* = w + scale·(w*−w), scale = cap/active;
2. пер-нейм кап (0.10/0.20/0.30) — water-filling `_cap_and_redistribute`
   (излишек перераспределяется на некапнутые пропорционально; кап
   инфизибелен при n·cap ≤ 1 → no-op).

**Views из скоров** — `views_from_scores`: каждый ненулевой Total даёт
абсолютный вью `Q = Total·0.5%` годовых с уверенностью
`c = clip(|Total|/6, 0.10, 0.95)`. `n_views=0` ⇒ posterior = чистый π
(функция весов и Σ, НЕ прогноз — потребители обязаны это помнить, F-11).

## 16. Факторная декомпозиция дисперсии — `factor_decomposition.py`

```
σ²_port = wᵀ(B·F·Bᵀ + D)w = bᵀFb + wᵀDw,   b = Bᵀw   (портфельные беты)
c_f = b_f·(F·b)_f                                    Euler-вклад фактора f
Σ_f c_f = bᵀFb (точно);  идиосинкратика = wᵀDw
identity_gap_pct = |wᵀΣw − (bᵀFb + wᵀDw)|/wᵀΣw·100   (float-шум ≈ 0, самопроверка)
```
Отвечает «откуда риск ПО ИСТОЧНИКАМ» (рыночная бета / стили / EM / ставки /
сырьё / специфика) — дополняет ERC (риск по активам). c_f может быть
отрицательным (фактор-хедж) — знак сохраняется.

**Marginal overlap:** «факторные двойники» — пары с систематической
корреляцией ≥0.90 через Σ_sys = B·F·Bᵀ (идиосинкратика исключена) и весом
≥2% каждая — одна факторная ставка куплена дважды.
`unique_risk_pct_i = d_i/(Σ_sys,ii + d_i)` — доля собственного риска актива.
**Атрибуция** `driven_by`: вклад актива в b_f равен w_i·B_if (Σ по активам
= b_f точно).
