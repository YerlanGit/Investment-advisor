### 1. Квант-ядро — 8.5/10
**Доказано:** EWMA⊕Ledoit-Wolf ковариация, Euler/MCTR (Σ=100% на live), Politis-Romano CI,
факторная декомпозиция (82.1+17.9≈100), 4-Pillar (F+V+T+C==total, max|Δ|=0.000), Growth×Cycle
режим с macro-overlay (GDP/безработица/инфляция, level⊕темп, ±0.05), BL (τ=0.05, Идзорек),
7 стресс-шоков с выпуклым капом, Base Currency USD + RFR 4.50% без mismatch, детерминизм.
Мультиколлинеарность диагностируется (κ=9.8, ортогонализация ON).
**Честный минус:** сигналы 4-Pillar/BL/идеи **не бэктестированы** — нет ни одного
исторического доказательства, что «Buy/Trim» добавляет альфу; продаётся как аналитика, не как
альфа — это корректно задисклеймлено, но для брокерского продукта это вопрос №1 от компетентного
клиента. Прокси-факторы в 2 из 7 стрессов (нет UUP/TIP).

### 2. Слой данных — 7.5/10
**Доказано:** per-source QualityGate (свежесть по каденции серии + sanity-диапазоны), CoVe на
24 строки — каждый инпут с методом и статусом, FX T-1 с покрытием, SEC-возраст 10-K
warn/stale-пороги, sparse-history дроп с раскрытием.
**Минусы:** цены только Tradernet (нет второго источника/кросс-чека котировок — при сбое брокера
слепнем целиком); CDS — HY-прокси, не настоящие спреды эмитентов; 730d окно ограничивает
длинные метрики; RAG-курация вручную.

### 3. Freedom API — 6/10 (главный разрыв ожиданий)
**Есть:** портфель (`get_portfolio`), user info, история через Tradernet, websocket-подписка,
Cloudflare-retry, ключи в Fernet-vault с ротацией MultiFernet, флоу подключения в боте.
**Нет (жёстко):** исполнения ордеров — «Action Plan» с ценами Buy/Sell/Stop и количествами
это **read-only советы**, пользователь исполняет руками; нет idempotent-ресинка позиций по
websocket в рантайме отчёта; нет обработки корпоративных действий (сплиты/дивиденды в PnL —
от брокера as-is); нет мульти-счетов. Для «advisor» это допустимо, но продукт называется
Action Plan — ожидание исполнения будет.

### 4. RAG + LLM — 7.5/10
**Есть:** ChromaDB (semantic 0.6 ⊕ recency 0.4), секционный чанкинг, метаданные
bank/section/tickers, boot-ingest fallback (обходит хрупкий Cloud Function/Eventarc), инвентарь
базы в CoVe, чекер «ИИ-цитирование» (память модели vs проверенные [RAG]-цитаты),
Structured Outputs (гарантированный JSON), prompt-caching, guard-фильтры (held/противоречия/
stress-cap/leverage-phrasing), grace-обрезка без обрывов мысли. Прод: 20 отчётов, 87 отрывков.
**Минусы:** курация KB — ручная (нет расписания обновлений, нет контроля прав на PDF банков —
юр.риск перепечатки выдержек!); стоимость DEEP на Opus ≈ $0.3–0.7/отчёт; латентность LLM
30–60 c; кросс-язычность retrieval (EN-чанки ↔ RU-запросы) не измерена.

### 5. Отчёты (Premium UI) — 8/10
**Доказано:** 0px overflow на 390/360px (Chromium), честные чекеры/легенды, leverage-правило
(маржа только при cash<0), signed URL 48ч, все секции согласованы (waterfall идентичен
BASE↔DEEP).
**Минусы:** «Скачать PDF» отдаёт HTML, а не PDF; только русский; тёмной темы нет; доступность
(a11y) не проверялась.

### 6. Telegram-бот — 6.5/10
**Есть:** аудированный флоу connect→analyze→report, WhitelistMiddleware (бета-гейт),
graceful shutdown без 409, PII-маскинг в логах, admin `/grant`.
**Минусы:** хендлеры aiogram **не покрыты e2e-тестами** (вся пирамида тестов — payload/математика);
нет per-user rate-limit (whitelist компенсирует только в бете); один админ; тексты ошибок местами
технические.

### 7. Биллинг — 4/10 (блокер монетизации)
**Есть:** честный токен-учёт на SQLite (gcsfuse-hardening: DELETE-journal, SIGKILL-тесты
атомарности), deferred-billing логика, admin-начисление.
**Нет:** платёжного шлюза ВООБЩЕ (ни Stars, ни карты, ни инвойсов) — выручка невозможна без
ручного вмешательства; нет тарифов/квот; SQLite на gcsfuse — осознанный компромисс, который
**навсегда фиксирует max-instances=1** (см. §9).

### 8. Безопасность / комплаенс — 6.5/10
**Есть:** брокер-ключи в Fernet (MultiFernet-ротация, H-7), маскинг PII, изолированный
ingest-бакет (M2: не reports!), hash-locked зависимости + blocking pip-audit (принятый
aiohttp-класс задокументирован), prompt-injection обёртки `<untrusted_data>` для RAG/брокер-данных,
дисклеймеры «не ИИР» во всех отчётах и CoVe.
**Минусы (строго):** дисклеймеры не проходили **юридическое ревью** (ЦБ РК/РФ — инвест-советник?);
права на банковские PDF в RAG не оформлены; секреты — env-vars Cloud Run (Secret Manager не
подтверждён); нет аудита действий админа; нет политики удаления данных пользователя (GDPR-like).

### 9. Инфра / CI/CD — 7/10
**Есть:** Cloud Build: build→test-в-образе→push→deploy(+фолбэк-ревизии Cloud Run), CI GitHub
(lock-consistency, pip-audit), деплой-гейт ловит реальные регрессии (доказано e24e8e9).
**Минусы:** **max-instances=1 · concurrency=1 · 1 CPU** — потолок ≈1 одновременный анализ;
DEEP занимает десятки секунд CPU+LLM → очередь при >5 активных пользователях; fail-soft шаг
Cloud Function маскирует сбои деплоя; staging-среды нет; канареек/автоотката по метрикам нет.

### 10. Наблюдаемость — 4/10
**Есть:** структурные логи (gatekeeper GK-коды, RAG boot-ingest, sync), CoVe как
«наблюдаемость данных» в самом продукте — это сильно; runbooks (RAG_TROUBLESHOOTING).
**Нет:** error-tracking (Sentry), метрик (латентность отчёта, токен-расход, LLM-ошибки),
алертов (упавший бот заметит только пользователь), SLA/SLO, дашборда здоровья.
