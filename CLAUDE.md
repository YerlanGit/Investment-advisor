Classic v3 Jinja (`src/templates/report_*_v3.html`) is the RETAINED auto-fallback (try/except in `render_report_html`) and is still test-pinned — set `PREMIUM_REPORT_ENABLED=false` to force it. The **Scenario** tier (2026-07-07) is a standalone deterministic report (0 LLM calls): `finance/scenario_report.build_scenario_payload` reuses `finance/scenario_engine` → `templates/report_scenario_v3.html` (its own Jinja; `html_renderer` routes `scenario` past Premium). See `REPORT_SECTIONS.md §11` + `docs/ROADMAP_SCENARIO_TIER.md`.

Same window as `SCENARIO_LOOKBACK_DAYS`. `days` is CALENDAR (`history.py: today − timedelta`); FX window bumped to 1900 to cover it. Analysis in `docs/ROADMAP_DATA_RESILIENCE.md §4`.

## Repository shape
- `src/` — bot (`tg_bot.py`, `entrypoint.py`; UI/routing only — series math lives in `finance/portfolio_series.py`), quant engine (`finance/`), report layer (`pdf_payload.py`, `premium_payload.py`, `premium_renderer.py`, `html_renderer.py`, `templates/`), LLM narrative (`ai_narrative.py`), advisory audit (`agent/gatekeeper.py`).
- `tests/` — pytest/unittest suites (`test_phase*.py`); validation surfaces reviewed alongside code changes.
- `cloud_function/` — GCS-triggered RAG ingest (ChromaDB build from bank PDFs). Two-bucket flow: PDFs land in **INBOX** `ramp-bot-chroma-db-inbox-investadv` (the Cloud Function trigger, **path A**); the built ChromaDB lives in **STORE** `ramp-bot-chroma-db-investadv` and the bot mirrors it on boot (`entrypoint._download_chroma_db`). **Reliable fallback (path B, 2026-07-04; incremental since 2026-07-09):** `entrypoint._boot_ingest_from_inbox` — on boot the bot ingests any INBOX PDFs **not yet in the store** in-container on a daemon thread (deps + pre-baked ONNX model + pinned chromadb present) and republishes to STORE, bypassing the fragile Cloud-Function/Eventarc/region chain. It self-heals BOTH an empty store AND incremental uploads (compares INBOX basenames vs. ingested `source`s), so the CoVe «база: N отчётов» count recovers on the next restart even when the Cloud Function silently failed — the root cause of «счётчик не обновился после 8 новых отчётов» (`RAG_BOOT_INGEST=0` to disable; `cloud_function/rag_engine.py` is kept identical to `src/agent/rag_engine.py`; the function's `chromadb` is pinned `==0.6.3`). `scripts/ingest_bank_report.py` — admin CLI for fast/manual PDF ingest (`--upload` publishes ChromaDB to STORE). RAG engine `src/agent/rag_engine.py` (chromadb imported lazily): section-aware size-bounded chunking + `bank`/`section`/`tickers` metadata + recency ranking + optional ticker filter. RAG observability (2026-07-04): `_fetch_rag_context` surfaces KB inventory (docs/chunks) live from `collection.count()`; CoVe carries a Bank-RAG inventory row + an `ИИ-цитирование банк-аналитики` checker that flags bank consensus the model emits from memory vs. verified `[RAG:file]` citations.
